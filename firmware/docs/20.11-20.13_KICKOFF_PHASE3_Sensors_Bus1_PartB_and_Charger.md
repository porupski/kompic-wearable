## PHASE 3 GOAL

Complete Bus 1 sensor coverage, then shift focus to **I2C bus contention** and power sequencing. Extract: MAX30101 FIFO drain timing, BME688 heater sequencing (the longest transaction on Bus 1), and BQ25619 ship-mode / boost control.

## WHAT YOU'LL PARSE

### 20.11 — MAX30101 Heart Rate & PPG
- I2C address (0x57 per v7.2)
- LED current control (RED, IR, 3rd LED if present)
- FIFO buffer (size, threshold interrupt, overflow)
- Interrupt register (INT_STAT, INT_EN; GPIO7 polarity)
- Data registers (FIFO_DATA, sample format)
- ADC resolution & sampling rate
- **Critical:** FIFO drain timing (how long to read full FIFO?)
- Power consumption (all modes + LED duty cycle)

### 20.12 — BME688 Environment (T/P/H/Gas)
- I2C address (0x76 per v7.2)
- Register map (CTRL_HUM, CTRL_MEAS, GAS_CNTRL, STATUS, DATA)
- Measurement modes (sleep, forced, sequential)
- Gas heater control (enable, temperature, duration) — **the long transaction**
- Conversion time (each measurement; gas heater especially)
- Heater settling time (how long resistance stabilizes?)
- BSEC algorithm reference (for gas → IAQ/VOC; v2+)
- **Critical:** Worst-case heater sequence timing (total I2C hold time)
- Power consumption (forced mode, gas heater active/inactive)

### 20.13 — BQ25619 Charger & Fuel Gauge
- I2C address (0x6A per v7.2; **Bus 2**, not Bus 1)
- Register map (STATUS, BATSNS, SYS_VOL, CHRGR_CTRL, SHIP_MODE_CTRL)
- Battery voltage & SoC readout (BATSNS accuracy, update rate)
- Charger states (pre-charge, CC, CV, done)
- Boost mode (PMID, 5V supply for MAX30101 LEDs / WS2812 / flashlight)
- **Boost vs. charge mutual exclusion** (firmware must arbitrate)
- Ship-mode entry (register write, timing to shutdown)
- QON button (dual-wired to GPIO16 per v7.2)
- INT polled via I2C (not a GPIO) — **no wake source from charger**
- Power consumption (charger active, standby, ship-mode)

## I2C CONTENTION DEEP-DIVE

Add a summary section comparing:
- **Bus 1 worst-case transaction:** BME688 gas heater (per AN001)
  - Typical config write: 2–3 ms
  - With heater: 30–100 ms hold (config, wait, re-read)
- **Bus 2 (isolated from Bus 1):** No contention risk; BQ reads <1 ms

Reference the system audit (§Sim 1) which modeled this. Your extraction confirms the numbers.

---

## KEY WATCHOUTS

1. **BME688 heater is the villain:** If firmware doesn't burst transactions, other sensors starve. Extract exact timing from AN001.
2. **MAX30101 FIFO threshold:** Set conservatively; FIFO drain is main latency tail on Bus 1.
3. **BQ25619 boost + charge:** Cannot both be on. Firmware must manage 5V rail arbitration.
