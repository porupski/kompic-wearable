## Summary

Core 1 LVGL orchestrator. Builds the main screen, settings tileview, alarm
screen and overlays; runs the 200 ms refresh task; fans out theme changes;
drains `g_ui_event_q`. The whole UI tree is registry-driven — adding a tile
means `#include "foo_tile.h"` + one row in `tile_registry.c`, nothing else.

Phase 6 v7.2 changes are entirely glue: the registry now points at the v7.2
chip drivers' tile headers, and a new ECG stub tile is registered at column 8.
No widget code changes, no overlay logic changes, no refresh-task changes.

## Pinout / bus from v7.2

N/A — UI layer. Display + touch pins owned by CO5300 / CST9217 drivers.

## Implementation

### `tile_registry.c`
Rewritten header comment block; +1 include; +1 entry. Body unchanged.

### `qvar_ecg/ecg_tile.h`
Now declares `extern const tile_desc_t ecg_tile_desc;` and re-includes
`tile_registry.h`. The legacy `ecg_tile_init()` entry point is preserved as
a no-op so any existing call sites (if any land later) won't break.

### `qvar_ecg/ecg_tile.c`
Three static functions implementing the tile_desc contract:
- `ecg_tile_build(lv_obj_t *parent)` — creates a single centred LVGL label
  with text `"ECG · Stub"`.
- `ecg_tile_refresh(void)` — no-op (no broker source yet).
- `ecg_tile_apply_theme(ui_theme_t theme)` — updates label text colour
  using the standard `COL_TEXT_LIGHT` / `COL_TEXT_DARK` macros.

### `qvar_ecg/CMakeLists.txt`
REQUIRES gains `lvgl` + `lvgl_ui`. Same pattern as every other v7.2 chip
component that ships a tile.

### `lvgl_ui/CMakeLists.txt`
REQUIRES list rewritten to the v7.2 part list; dead entries removed; ECG
added. PRIV_REQUIRES preserved.

## Test harness

`firmware/test/test_lvgl_ui.c` — boots lvgl_port + a minimal CO5300 stub,
calls `lvgl_ui_init(&cfg)`, walks the tile registry, asserts every entry has
a non-NULL `desc` with non-NULL `init` / `update` / `apply_theme` pointers,
then drives synthetic broker writes (battery + RTC) and confirms the main
screen labels refresh inside one 200 ms tick.

Caveats noted in the test header: without a real display the LVGL output goes
to a software-only framebuffer (`lv_disp_drv_register` with a memory display).
This validates the tile graph and refresh ordering, **not** pixel output —
that requires the real CO5300 + Phase 0 boot path.

### Notes
- The refresh task already calls `lvgl_port_lock(10 ms timeout)` — short
  enough to avoid blocking the LVGL render task on touch events.
- Theme fan-out is registry-driven (zero per-tile code in lvgl_ui.c). The
  ECG tile inherits this automatically.
- The 10th tile (ECG) is at column 8 (s_tiles index 9 because `health_tile`
  is at index 0 / column -1). The settings tileview default-start column
  stays at 2 (System), so adding ECG at the right edge has no UX impact on
  existing users.

## Defects discovered

[DEFECT-001] ECG tile shows placeholder label, no waveform | components/qvar_ecg/ecg_tile.c | LOW | The tile is registered and renders `"ECG · Stub"` but does no broker reading. This is intentional — Qvar register map is unverified against the LSM6DSV16X datasheet, so wiring waveform samples through would mean shipping unsubstantiated data. Disposition: Phase 20 (datasheet pass) implements the real ECG broker slot and rewires this tile.

[DEFECT-002] No LED-strobe toggle control in alarm-edit overlay | components/alarm/alarm_tile.c | LOW | (Tracked here because the UI fix lands in lvgl_ui's domain.) The alarm slot's `led_strobe` bit is persisted but the overlay currently exposes only HH/MM/pattern rollers. Disposition: small follow-up — add a single icon-toggle button below the pattern roller.

[DEFECT-003] `light_tile_create_overlay()` declaration source-of-truth not annotated in lvgl_ui.c | components/lvgl_ui/lvgl_ui.c:61 | LOW | The symbol resolves to `components/veml6030/light_tile.h` via the REQUIRES chain. A short comment at the include site would make the dependency obvious to a reader of lvgl_ui.c. Disposition: cosmetic, fold into next lvgl_ui touch.
