## PHASE 2 GOAL

Extract **sensor hardware specifications** — I2C addresses, register maps, interrupt behavior, power states, and timing — for the first half of the sensor bus (Bus 1, GPIO1/2 @400kHz). Focus on initialization sequences, data read paths, interrupt latency, and power budgets.

## WHAT YOU'LL PARSE

### 20.06 — VEML6030 Light Sensor
- I2C address (0x10 per v7.2)
- Register map: ALS_CONF, ALS, POWER_SAVING, INT_CMD, INT_THRESHOLD
- Integration time & gain settings (range, sensitivity curves)
- Interrupt generation (threshold-based)
- Auto-ranging capability
- Power consumption (all modes)

### 20.07 — LIS3MDLTR Magnetometer
- I2C address (0x1C per v7.2)
- Register map: CTRL_REG1–5, STATUS, MAG_OUT_X/Y/Z, INT_CFG, INT_SRC
- Output data range (±4 gauss per v7.2)
- Sensitivity curve (mG/LSB per range)
- Sampling rate options
- Hard-iron calibration storage (NVS)
- **LRA magnet offset impact** [CRITICAL — see v7.2]
- Power consumption (all modes)

### 20.08 — MAX-M10S GPS Receiver
- UART1 interface (GPIO17/18 per v7.2)
- Baud rate (default & configurable)
- NMEA sentence formats (GGA, RMC, VTG)
- UBX binary protocol (NAV-PVT, NAV-TIMEUTC, NAV-STATUS)
- **Time-before-fix capability** [CRITICAL — see audit §GPS time gated on fix]
- 1PPS output (GPIO46 per v7.2; accuracy)
- Backup power supercap specs (0.22F, 2.7V per v7.2)
- Startup time (cold, warm, hot)
- Power consumption (acquisition, tracking, sleep)

### 20.09 — TMP117 Skin Temperature Sensor
- I2C address (0x48 per v7.2)
- Register map: TEMP, CONF, T_HIGH, T_LOW, EEPROM
- Temperature range & accuracy (±0.1°C typical)
- Resolution & conversion time
- Interrupt generation (high/low threshold)
- Thermal time constant
- Power consumption (all states)

### 20.10 — LSM6DSV16X IMU (Accel + Gyro)
- I2C address (0x6B per v7.2; WHO_AM_I = 0x70)
- Register map (CTRL1_XL, CTRL2_G, CTRL3_C, INT1_CTRL, FIFO_CTRL, STATUS_REG)
- Accel range (±2g, ±4g, ±8g, ±16g per v7.2)
- Gyro range (±125, ±250, ±500, ±1000, ±2000 dps)
- Sensitivity (mg/LSB, mdps/LSB per range)
- Sampling rates (6.5 Hz to 6.7 kHz)
- FIFO buffer (size, threshold, overflow behavior)
- INT1 pin (raise-to-wake config, GPIO8 polarity, latency)
- Qvar block (reference only for now; v2+ ECG)
- Machine Learning Core (MLC) — reference only
- Power consumption (accel, gyro, both, sleep)

## KEY WATCHOUTS

1. **LIS3MDLTR + LRA magnet:** v7.2 flags ~1G DC offset from LRA magnet. Datasheet won't mention this; note it as a known bias.
2. **MAX-M10S time-before-fix:** Audit flagged GPS time gated on full fix. Extract exact NMEA vs. UBX timing.
3. **LSM6DSV16X is huge:** Focus on accel/gyro/INT1 for this phase; ignore MLC details.
4. **VEML6030 auto-ranging:** Confirm whether automatic or manual; affects auto-brightness algorithm.
