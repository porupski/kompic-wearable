## WHAT YOU'LL PARSE

### Chip 20.01 — CO5300 AMOLED Display Driver

**File:** `CO_5300_Datasheet_V0_00_20230328_07edb82936.pdf` (~9 MB)  
**Focus areas:**
- **Pinout & signal names:** QSPI (D0–D3, CLK, CS), TE, RST, power rails
- **QSPI command protocol:** Instruction vs. pixel mode, addressing, byte order
- **COLMOD register (0x3A):** Mandatory 0x77 (RGB888) per v7.2; confirm no other modes are viable
- **Sleep/wake sequence:** DISPOFF → SLPIN → wake → SLPOUT → DISPON (exact register writes + timing)
- **Brightness control:** Register 0x51 range, units, default
- **TE (tearing) pin:** Polarity, timing relative to QSPI writes
- **Power consumption:** By mode (sleep, active, partial update)

**Key questions:**
- Does the datasheet confirm RGB888 (0x77) is the only sane mode for QSPI?
- What is the minimum TE period (how fast can we pump pixels)?
- Is there any undocumented "fast sleep" mode?

**What to extract:**
- Full QSPI command set (focus on: 0x02 instruction, 0x32 pixel, 0x28 DISPOFF, 0x10 SLPIN, 0x11 SLPOUT, 0x29 DISPON, 0x51 brightness)
- All display-control registers (COLMOD, BRIGHTNESS, SLEEP commands)
- Timing budgets for TE, SLPIN/SLPOUT, frame update
- Power consumption table (all states)

### Chip 20.03 — ESP32-S3 MCU

**Files:**
- `esp32-s3_datasheet_en.pdf`
- `esp32-s3_technical_reference_manual_en.pdf`
- `esp32-s3-wroom-1_wroom-1u_datasheet_en.pdf`

**Focus areas:**
- **GPIO layout:** All 48 usable pins, their functions, strapping constraints
- **I2C (I2C0 / I2C1 / new i2c_master API):** Timing, clock speeds, pull-ups, open-drain
- **SPI2 / FSPI (QSPI):** Pin assignments, clock speeds, DMA, IOMUX
- **UART1:** GPIO17/18, baud rate limits
- **I2S0 PDM:** GPIO47/48, clock source, sample rate
- **RMT (for WS2812):** GPIO42 assignment, timing constraints
- **LEDC:** GPIO41 PWM, frequency range
- **Deep sleep / light sleep:** Wake sources, current draw, RTC GPIO
- **Boot strapping:** GPIO0, GPIO2, GPIO45, GPIO46 constraints (per v7.2 warnings)
- **PSRAM:** Octal mode, 80 MHz clock, capacity (8 MB)
- **USB CDC:** GPIO19/20, native support

**Key questions:**
- What is the absolute minimum current draw in light-sleep with RTC running?
- Are there undocumented errata on QSPI performance or PSRAM reliability?
- What is the exact IOMUX configuration for SPI2 vs. other peripherals?

**What to extract:**
- Master GPIO assignment table (cross-check against v7.2 Table)
- I2C timing & pull-up specs
- QSPI clock limits & DMA capabilities
- Sleep current figures (deep, light, with/without RTC)
- Boot-strap pin constraints (especially GPIO0, 45, 46)

### Chip 20.04 — PCF85063A RTC

**File:** `PCF85063A.pdf` (~700 KB)  
**Focus areas:**
- **I2C address:** 0x51 (bus 1)
- **Register map:** Control, time/date, alarm, interrupt, clocking
- **INT pin:** Polarity, trigger conditions (alarm, periodic interrupt)
- **CLKOUT:** Optional 32 kHz output (JP5 config per v7.2)
- **Backup power:** ML621 cell specs, charging circuit, timekeeping duration
- **Power consumption:** Active vs. backup mode, backup cell current draw
- **Time precision:** Drift specs, 32 kHz crystal requirements

**Key questions:**
- Does the datasheet specify maximum backup timekeeping duration on the ML621?
- What is the exact timing of the 1 Hz output for RTC discipline (1PPS)?

**What to extract:**
- Full register map (time/date format, alarm registers, control bits)
- I2C protocol & timing
- Backup battery charging curve & current limits
- Crystal frequency tolerance and load capacitance
- Power consumption (active, backup, off)

### Chip 20.05 — TPS62840 3.3V Buck Converter

**File:** `tps62840.pdf` (~3.1 MB)  
**Focus areas:**
- **Input/output voltages:** SYS (3.4–4.2V) → 3.2V (per v7.2 100kΩ divider)
- **Output current limit:** 750 mA rated
- **Soft-start & ramp timing:** How fast does it come up?
- **Enable pin (EN):** Active-high, pulled to SYS (always on with battery)
- **Mode control (JP15):** PFM vs. PWM, efficiency curves
- **Switching frequency:** Default and if configurable
- **Power consumption (light load, sleep mode):** Quiescent current
- **Thermal management:** Operating range, derating

**Key questions:**
- Is the 100kΩ divider in the datasheet, or is this a custom v7.2 choice?
- What is the quiescent current in PFM mode (what we expect to use)?
- Does the converter have a "super low-power" mode for light-sleep?

**What to extract:**
- Functional block diagram (feedback network, switching loop)
- VSET divider equation (confirm 3.2V output setting)
- Power supply sequencing (soft-start, ramp rate)
- Efficiency curve (load current vs. power loss)
- Quiescent current (active, mode-dependent)
